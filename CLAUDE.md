# modern-xlsx

XLSX read/write library: Rust WASM core + TypeScript API.

## Project Structure

```
crates/modern-xlsx-core/     # Rust core (OOXML parsing/writing, ZIP, XML)
crates/modern-xlsx-wasm/     # WASM bridge (wasm-bindgen exports)
packages/modern-xlsx/        # TypeScript package (public API, npm)
  src/                       # Source: index.ts, workbook.ts, types.ts, etc.
  wasm/                      # Built WASM artifacts (generated by wasm-pack)
  __tests__/                 # Vitest integration tests
  dist/                      # Built JS output (generated by tsdown)
docs/plans/                  # Design and implementation plans
```

## Build Commands

```bash
# Rust tests (217 tests)
cargo test -p modern-xlsx-core

# WASM build (from repo root)
cd crates/modern-xlsx-wasm && wasm-pack build --target web --release --out-dir ../../packages/modern-xlsx/wasm --no-opt

# TypeScript build
pnpm -C packages/modern-xlsx build

# TypeScript tests (487 tests)
pnpm -C packages/modern-xlsx test

# Lint & format
pnpm -C packages/modern-xlsx lint
cargo clippy -p modern-xlsx-core -- -D warnings

# Type check
pnpm -C packages/modern-xlsx typecheck
```

## Architecture

- **Reader path:** `Uint8Array` -> WASM `read()` -> ZIP decompress -> SAX-parse XML parts -> `WorkbookData` (JSON) -> JS
- **Writer path:** JS `WorkbookData` -> WASM `write()` -> build SST inline (`to_xml_with_sst`) -> ZIP compress -> `Uint8Array`
- **SST behavior:** Writer expects `cellType: "sharedString"` with actual text in `value`. The writer builds the shared string table and remaps indices inline during XML generation (no worksheet clone).
- **JSON bridge:** Data crosses the WASM boundary as JSON strings (serde_json + JSON.parse) — 8-13x faster than serde_wasm_bindgen for large workbooks.
- **Serde:** All Rust types use `#[serde(rename_all = "camelCase")]` for JS interop.

## Toolchain Versions

- Rust 1.94.0 Edition 2024 (CI: dtolnay/rust-toolchain@1.94.0), wasm-bindgen 0.2.114, quick-xml 0.39.2, zip 8.1 (`deflate`)
- TypeScript 6.0.0-dev, Vitest 4.1.0-beta.5, tsdown 0.21.0-beta.2, Biome 2.4.4, pnpm 11

## Code Conventions

- **Rust:** `thiserror` for errors, `quick-xml` SAX parsing, co-located `#[cfg(test)]` modules, `std::mem::take` for buffer reuse
- **TypeScript:** ESM-only, single quotes, trailing commas, semicolons, 2-space indent (Biome)
- **WASM bridge:** JSON string serialization for data, `JsError` for error conversion
